# 들어가며

이 책은 RuStudio 학습 시리즈의 일곱 번째 책이다.

지금까지 모든 입력은 두 종류였다.

```text
1. cpal 입력 (마이크/입력 장치)
2. 코드로 합성한 신호 (oscillator + envelope 등)
```

이 책은 세 번째 입력 경로 — **파일** — 을 다룬다.

```text
input  : .wav, .mp3, .flac, .ogg, ... 파일
output : .wav (가장 흔함), 그 외 인코딩 포맷
```

또한 출력 경로 — 처리 결과를 파일로 저장하는 일 — 도 함께 본다. 이 둘을 합치면 **오프라인 처리**라는 새로운 사고방식이 들어온다.

## 실시간 vs 오프라인

지금까지의 흐름은 전부 실시간이었다.

```text
실시간:
  cpal 콜백이 매 ms마다 깨어남
  → 그 시간 안에 결과를 만들어야 함
  → 처리 시간 < buffer 시간
```

오프라인은 다르다.

```text
오프라인:
  파일 → 메모리 → 처리 → 파일
  → 시간 제약 없음
  → 처리가 1초든 1시간이든 기능적으로 같은 결과
  → "지금 이 순간"이 아닌 "곡 전체"를 알 수 있음
```

이 차이가 mastering의 두 모드(실시간 모니터 vs 곡 발송용 처리)와 정확히 일치한다.

## 이 책이 다루는 것

```text
1. 파일 → 신호의 흐름 — decode 파이프라인
2. WAV 포맷 — 정수 PCM, float, 인터리브, 샘플 포맷 변환
3. 압축 포맷 — symphonia로 MP3/FLAC/OGG 디코딩
4. 샘플레이트 변환 — rubato로 48k ↔ 44.1k 같은 변환
5. 신호 → 파일 — encoder의 위치
6. 오프라인 vs 실시간 처리 경로의 분리
```

## 이 책이 다루지 않는 것

```text
✗ 새 인코더 작성 (mp3/aac 인코딩 직접 구현)
✗ 비디오 컨테이너 (mp4, mov)
✗ 라이선스 이슈 깊이 (특히 mp3/aac/codec 특허)
✗ 스트리밍 프로토콜 (HLS, DASH, etc.)
✗ DRM
```

특히 mp3 인코딩은 라이선스 측면이 까다로워 직접 구현하지 않는다. 디코딩은 OS/symphonia 같은 라이브러리로 한다.

## 핵심 crate

```text
hound      : WAV read/write (단순, 학습용 최고)
symphonia  : MP3/FLAC/OGG/WAV 등 다양한 포맷 디코딩
rubato     : 고품질 샘플레이트 변환 (FFT-based 또는 polynomial-based)
```

## 이 책을 다 읽고 나면

- WAV 파일이 메모리에 어떤 모양으로 풀리는지 그릴 수 있다.
- int16 PCM과 f32 PCM의 변환 식을 안다.
- interleaved와 planar 둘 사이를 코드로 변환할 수 있다.
- mp3/flac 같은 압축 포맷의 디코딩이 packet 단위라는 점을 안다.
- 48 kHz와 44.1 kHz 사이를 단순 인덱스 스킵으로 못 바꾸는 이유를 설명할 수 있다.
- 파일 → 처리 → 파일의 오프라인 파이프라인을 그릴 수 있다.
- 같은 처리 코드를 실시간과 오프라인 양쪽에서 쓸 때의 분리 원칙을 안다.
