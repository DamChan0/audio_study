# 이 책을 읽는 방법

이 mdBook은 DSP 수식 사전이 아니다.

목표는 다음 두 가지다.

- `Phase 2 - DSP 블록`을 스스로 구현할 수 있게 만들기
- 구현한 블록을 `cpal` 콜백에 자신 있게 삽입할 수 있게 만들기

## 추천 독서 순서

1. 이 장에서 전체 흐름을 잡는다.
2. `DSP가 cpal 위에서 푸는 문제`로 왜 이 Phase가 필요한지 정리한다.
3. **시간 도메인 블록** (오실레이터 → 엔벨로프 → PolyBLEP)을 먼저 본다.
4. **주파수 도메인 블록** (Biquad → EQ → FFT)을 그 다음에 본다.
5. 측정 블록 (LUFS)은 마지막에 본다. 앞선 블록이 다 있어야 의미가 생긴다.

## 이 책에서 예제를 읽는 방법

예제 스텁은 `todo!()` 로 비어 있다. 완성 코드를 제공하지 않는다.

예제를 읽고 구현할 때는 항상 아래 순서로 한다.

1. 이 블록이 해결하는 문제가 무엇인지 한 문장으로 쓴다.
2. **입력/출력 시그니처**가 이미 정해져 있다. 왜 그 타입인지 먼저 답한다.
3. 콜백 안에서 할 일 / 콜백 밖에서 할 일을 구분한다.
4. `todo!()` 를 채운다. (한 번에 한 블록만, 다른 곳은 신경 쓰지 않는다)
5. `cargo run --bin <name>` 으로 소리로 확인한다.
6. 이 블록이 `mod_player` / `mod_mastering` 에 어떻게 들어갈지 한 줄로 적는다.

## 학습 완료 기준

이 책을 다 읽고 나면 적어도 아래 질문에는 바로 답할 수 있어야 한다.

- 위상 누산을 `f32`로 하면 어떤 시점에서 정밀도 문제가 생기는가?
- ADSR Release 중에 Note On이 다시 들어오면 상태 전이는 어떻게 되는가?
- Saw wave를 단순 구현하면 왜 고주파에서 지저분해지는가?
- Biquad의 `a` / `b` 계수를 갑자기 바꾸면 왜 클릭이 생기는가?
- Peaking EQ를 8개 직렬 연결하면 위상 응답은 어떻게 되는가?
- FFT window를 직사각형으로 두면 어떤 artifact가 생기는가?
- LUFS Integrated 값이 Momentary / Short-term과 다른 이유는 무엇인가?

이 질문들에 자연스럽게 답할 수 있으면, Phase 3 (파일 I/O) 또는 Phase 4 (오디오 그래프)로 넘어갈 준비가 된 것이다.
